# 🚀 AI工具集成平台

基于 **Next.js 14** 和 **飞书多维表格** 构建的现代化AI工具集成平台，为用户提供一站式AI服务体验。

## ✨ 项目特色

- 🔐 **智能认证系统** - 基于飞书多维表格的用户管理
- 📰 **AI新闻聚合** - 实时获取最新AI行业动态  
- 🛠️ **AI工具集成** - 解决AI使用痛点的工具箱
- 📱 **移动端优先** - 完美适配手机和桌面端
- 🌐 **跨端部署** - 支持Web应用和微信小程序

## 🎯 核心功能模块

### 1. 🏠 智能仪表板
- **个性化推荐**: 基于用户使用习惯推荐AI工具和新闻
- **使用统计**: 直观显示工具使用情况和数据分析
- **快捷入口**: 一键访问常用AI工具

### 2. 📰 AI新闻中心  
解决AI信息分散的痛点：
- **实时聚合**: 自动抓取OpenAI、Google、微软等官方动态
- **智能分类**: 技术更新 | 产品发布 | 行业动态 | 政策法规
- **热度排序**: 基于用户关注度和影响力排序
- **个人收藏**: 保存感兴趣的内容

### 3. 🔧 AI工具箱
集成解决常见AI使用痛点的工具：

#### 🧠 思维导图生成器
- **功能**: 将文本内容转换为可视化思维导图
- **痛点解决**: 信息整理和可视化困难
- **特色**: 支持中英文、自动提取关键信息、多种导出格式

#### 💬 AI对话助手
- **功能**: 集成多种大模型(GPT、Claude、Gemini)的统一对话界面
- **痛点解决**: 多平台切换、账号管理复杂
- **特色**: 一个界面调用多个模型、对话历史管理

#### ✍️ 智能写作助手
- **功能**: 文档生成、内容优化、格式转换
- **痛点解决**: 写作效率低、格式化困难
- **特色**: 模板库、批量处理、多种文档格式

#### 💻 代码生成器
- **功能**: 自然语言转代码、代码优化建议
- **痛点解决**: 编程学习成本高、重复代码编写
- **特色**: 支持多语言、代码解释、最佳实践推荐

#### 🖼️ 图片处理工具
- **功能**: AI图片生成、编辑、优化
- **痛点解决**: 设计成本高、处理工具分散
- **特色**: 一键生成、批量处理、格式转换

#### 📝 Prompt模板库
- **功能**: 精选提示词模板、社区分享
- **痛点解决**: Prompt编写困难、效果不佳
- **特色**: 分类整理、效果评分、用户贡献

### 4. 👤 个人工作区
- **历史记录**: 所有工具使用记录和结果保存
- **个人模板**: 自定义prompt和工具配置
- **使用分析**: 个人AI使用习惯和效率统计
- **收藏管理**: 新闻、模板、结果统一收藏

## 🏗️ 技术架构

### 前端技术栈
```typescript
- Next.js 14 (App Router)     // React全栈框架
- TypeScript 5.3              // 类型安全
- Tailwind CSS 3.3            // 原子化CSS
- Zustand                     // 状态管理
- React Hook Form             // 表单处理
- Chart.js                    // 数据可视化
```

### 后端架构
```typescript
- Next.js API Routes          // 服务端API
- 飞书多维表格                 // 数据存储
- WebSocket                   // 实时通信
- 第三方AI API                // OpenAI、Claude等
```

### 部署方案
```bash
- Vercel                      // 前端部署
- GitHub                      // 代码仓库
- 飞书开放平台                 // 数据后端
```

## 📊 数据表设计

### 核心数据表结构

#### 用户表 (已有)
```sql
- 用户ID, 姓名, 手机号, 用户类型, 注册时间, 最后登录时间
```

#### AI新闻表 (新增)
```sql
- 新闻ID, 标题, 摘要, 来源, 发布时间, 分类, 链接, 是否热门
```

#### 工具使用记录表 (新增)  
```sql
- 记录ID, 用户ID, 工具名称, 工具类型, 使用时间, 输入内容, 输出结果
```

#### 用户收藏表 (新增)
```sql
- 收藏ID, 用户ID, 内容类型, 内容ID, 收藏时间, 备注
```

#### Prompt模板表 (新增)
```sql
- 模板ID, 用户ID, 模板名称, 内容, 分类, 是否公开, 使用次数, 评分
```

## 📱 移动端优化

- **懒加载**: 图片和内容按需加载
- **缓存策略**: 智能缓存常用数据
- **压缩优化**: 资源压缩和CDN分发

## 🔧 开发环境设置

### 环境要求
- Node.js 18.0+
- npm 9.0+
- 飞书开发者账号

### 安装依赖
```bash
npm install
```

### 配置环境变量
```bash
cp env.template .env.local
# 编辑 .env.local 填入飞书应用配置
```

### 启动开发服务器
```bash
npm run dev
```

## 📚 使用指南

### 飞书应用配置
1. 访问 [飞书开放平台](https://open.feishu.cn/app)
2. 创建新应用，获取 App ID 和 App Secret
3. 配置权限：`bitable:app`、`bitable:app:readonly`
4. 创建多维表格，配置数据表结构

### 数据表创建
参考 `env.template` 中的详细字段说明创建各个数据表

### API接口
```typescript
GET  /api/news/latest           // 获取最新AI新闻
POST /api/tools/mindmap         // 生成思维导图
POST /api/tools/chat            // AI对话
POST /api/auth                  // 用户认证
```

## 🌟 项目亮点

### 解决的核心痛点
1. **信息获取痛点**: AI新闻分散 → 一站式新闻聚合
2. **工具使用痛点**: 多平台切换 → 统一工具界面  
3. **效率问题**: 重复性工作 → 智能模板和批处理
4. **学习成本**: 不知如何使用AI → 内置教程和最佳实践
5. **移动体验痛点**: 网页不适配手机 → 移动端优先设计

### 技术创新
- **飞书表格作为数据库**: 低成本、高可用的数据存储方案
- **AI工具聚合**: 统一接口调用多个AI服务
- **智能推荐系统**: 基于用户行为的个性化推荐
- **跨端一体化**: Web + 小程序无缝体验
